# Shared OpenDog Issue Index

Use this index for issues that should be visible across projects. Keep project-specific evidence in `reports/<project>/`, then link it here with a stable issue id.

## Rules

- One-to-one routing: OpenDog responses to project-specific reports stay under that project's `reports/<project>/` directory or in the report's `OpenDog response` section.
- Shared visibility: any issue that may affect more than one project gets an `ODX-YYYYMMDD-<slug>` id here.
- Resolution visibility: when OpenDog fixes, defers, or rejects an issue, update both this shared index and the originating project report.
